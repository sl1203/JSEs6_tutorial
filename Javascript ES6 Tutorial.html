What Are Arrow Functions?

Arrow functions  =>, are a more concise syntax for writing function expressions. They are similar to lambda function in Python, and offer a concise syntax.  With arrow function, we don't have to use function and return keywords (itâ€™s implicit in arrow functions).

Example:

Basic Syntax with Multiple Parameters 


// ES5
var add = function(x, y) {
  return x + y;
};

// ES6
const addES6 = (x, y) => x + y ;

It's simple and concise!!! However, if there are multiple lines of codes within the arrow function, we still have to use function and, return keywords.

Arrow Functions and Map, Filter, & Reduce (function of array)



Consider the following data:

data = [
  {
    name: 'John',
    age: 6,
    gender: 'M',
    grade: 1
  },
  {
    name: 'Lizzy',
    age: 16,
    gender: 'F',
    grade: 10
  },
  {
    name: 'David',
    age: 18,

    gender: 'M',

    grade: 12
  },
  {
    name: 'Joey',
    age: 10,
    gender: 'M',

    grade: 5
  }
];

As you can see we have an array of objects. Each object represents a student, with a name, an age, and a grade.


Our filter method only return 'F' (Female) student:

let femaleStudents= data.filter((student) => {
  return student.gender === 'F';
})

Or we can simplify further:

let femaleStudents= data.filter((student) => student.gender === 'F');

Or we can filter based on the age:

data.filter((student) => student.age > 10 ); // return Lizzy and David


Now that we can list all the names of students.

 data.map((student) => student.name); // return ["John", "Lizzy", "David", "Joey"]

Sum the ages of students: (use reduce)

data.reduce((sum, student) => {
  return sum + student.age;
}, 0); //return 50


Template Literals:

 From MDN: Template literals are string literals allowing embedded expressions. 
Template literals are enclosed by the back-tick (` `) character instead of double or single quotes. Template literals can contain placeholders. These are indicated by the dollar sign and curly braces (${expression}).

For exmple: 

var firstName='John';
var lastName ='Doe';

var name=`Full name is ${firstName} ${lastName}`;
console.log(name) //Full name is John Doe


Destructuring Assignment:

According to MDN: The destructuring assignment syntax is a JavaScript expression that makes it possible to unpack values from arrays, or properties from objects, into distinct variables.

For example: 

var a, b
[a,b]=[10,20] //a=10, b=20

[b,a]= [a,b] // b was assigned to 10, a=20
[a, b, ...rest] = [10, 20, 30, 40, 50]; // rest was an array, rest=[30, 40, 50]


Var, let or const

ES6 introduced  two new ways of creating variables: let and const. When a viable was created in JS, it was assigned the initial value 'undefined'. 

A variable created by using var can be updated and redefined. However, a variable created by using  let can be updated, but NOT redefined. A variable created by using  const can NOT be updated, or redefined.

The following codes show the difference between the var, const and let.

//Var
//define

var num1 =4;
console.log(num1); //return 4
//update
num1= 10; // success! num1=10
console.log(num1) 

//redefine
var num1='orange';
console.log(num1); //return "orange"



//let (can define and update, NO REDEFINE)

let num2 =4 ;
console.log(num2); //return 4
//update
num2=100; 
console.log(num2); //return 100

let num2='orange';
console.log(num2); //ERR: can not redefine



//const (Can define, can't update and redefine)
//define
const num3 =5;
console.log(num3); //return 5
//update
num3=100;
console.log(num3); //ERR: can't update

//redefine
const num3='orange' ; //ERR: can't redefine
console.log(num3);
